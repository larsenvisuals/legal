Privacy Policy

AliceClient · Larsen Visuals LLC · Effective June 17, 2026

AliceClient is a private, single-user internal tool operated by Larsen Visuals LLC (“Company”). It is used solely by the Company’s authorized operator to administer the Company’s own business. It is not a public product and has no external users.

1. What data we access

When the operator connects a third-party account (such as Intuit QuickBooks Online), the Software accesses the Company’s own business data — for example invoices, bills, customer names, and account balances — on a read-only basis, solely to display that data back to the operator.

The Software may also access the operator’s own Apple financial data — transactions, balances, and account details for Apple Card, Apple Cash, and Savings — through Apple’s FinanceKit framework on a read-only basis. This data is read on the operator’s device, only after the operator grants explicit FinanceKit authorization, and is used solely to display and analyze the operator’s own spending (balances, spending trends, and anomaly flagging) back to the operator. The operator can revoke this authorization at any time in iOS Settings.

2. How we use it

Accessed data is used only for the Company’s internal financial review and administration. It is never used for advertising, profiling, or any purpose unrelated to operating the Company’s business.

3. Storage and sharing

Data is stored locally on hardware owned and controlled by Larsen Visuals LLC. It is not sold, rented, or disclosed to any third party for that party’s own purposes, and it is not placed into any advertising, analytics, or data-broker service. The only external processing is the generative-AI processing described in Section 4, performed by a provider acting on the Company’s behalf.

4. Generative AI processing

Some assistant features summarize, or answer the operator’s questions about, the Company’s own data using a third-party generative-AI provider (Anthropic, via its API). When used, data is sent to that provider only to generate a result that is presented back to the operator. The provider acts as a data processor on the Company’s behalf, does not use the data to train its models, and the data is not used for any other purpose or disclosed onward. The invoice and accounting display features do not send data to the AI provider; only the assistant and summary features do, and only as needed to serve the operator.

5. Access scope and retention

The Software requests the minimum access required (read-only) and retains a local cache of the operator’s own data only as long as useful for review. The operator may disconnect the integration or delete the local data at any time, which removes further access.

6. Security

Access credentials are stored locally with restricted permissions and are never exposed to third parties.
